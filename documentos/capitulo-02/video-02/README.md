# Video 02-02

## Objetivo de aprendizaje
Diseñar un esquema inicial de base de datos para un gestor documental en SQL Server, modelando entidades principales, relaciones y reglas de integridad.

## Que aprende el estudiante
- Definir tablas nucleares de un dominio documental: usuarios, documentos, categorias, etiquetas, versiones y permisos.
- Establecer claves primarias y foraneas para garantizar integridad referencial.
- Crear indices iniciales para optimizar consultas comunes de busqueda, filtrado y trazabilidad.
- Incorporar restricciones de negocio con CHECK, UNIQUE y valores por defecto.

## Prompt trabajado
Diseña las tablas de un gestor documental (usuarios, documentos, categorías, etiquetas, versiones y permisos) y genera claves primarias, foráneas e índices iniciales.

## Evidencias generadas
- codigo/01-esquema-gestor-documental.sql
- prompts/01-disena-tablas-gestor-documental.md

## Resultado esperado
El estudiante obtiene un script ejecutable que crea la base de datos y su modelo relacional inicial, listo para evolucionar con datos de prueba, consultas y controles de acceso mas avanzados.
